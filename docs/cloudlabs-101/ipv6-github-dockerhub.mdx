# Access GitHub & Docker Hub from IPv6-only VPS

VirtualLabs and some IPv6-only VPS instances can't directly reach services that are still IPv4-only, such as **GitHub** and **Docker Hub**. This tutorial provides practical solutions so you can still `git clone`, `git push`, `docker pull`, and `docker login` from a VPS that only has an IPv6 address.

## The Problem

When you run:

```bash
git clone https://github.com/user/repo.git
# or
docker pull nginx:latest
```

You might get errors like:

```
fatal: unable to access 'https://github.com/...': Could not resolve host: github.com
```

```
Error response from daemon: Get https://registry-1.docker.io/v2/: dial tcp: lookup registry-1.docker.io: no such host
```

This happens because **GitHub**, **Docker Hub**, and many developer services still don't support direct IPv6 connections (as of June 2026). Your VPS only has IPv6 connectivity, so DNS resolution fails.

## Solution 1: GitHub via IPv6 Proxy (/etc/hosts)

You can route traffic to GitHub through an **IPv6 proxy** — a server that accepts IPv6 connections and forwards them to GitHub over IPv4. The method: add mappings to `/etc/hosts`.

### Option A: TransIP GitHub IPv6 Proxy

Best for VPS instances on TransIP / SandboxVPS.

```bash title="/etc/hosts"
2a01:7c8:7c8::1337 github.com
2a01:7c8:7c8::1337 api.github.com
2a01:7c8:7c8::1337 codeload.github.com
2a01:7c8:7c8::1337 objects.githubusercontent.com
2a01:7c8:7c8::1337 raw.githubusercontent.com
```

### Option B: DanWin1210 Public IPv6 Proxy

A public proxy usable from anywhere. More comprehensive — includes GitHub Packages, GHCR, and GitHub Assets.

```bash title="/etc/hosts"
# GitHub core services
2a01:4f8:c010:d56::2 github.com
2a01:4f8:c010:d56::3 api.github.com
2a01:4f8:c010:d56::4 codeload.github.com
# GitHub Container Registry
2a01:4f8:c010:d56::6 ghcr.io
# GitHub Packages (npm, maven, nuget, rubygems)
2a01:4f8:c010:d56::7 pkg.github.com npm.pkg.github.com maven.pkg.github.com nuget.pkg.github.com rubygems.pkg.github.com
# GitHub Uploads
2a01:4f8:c010:d56::8 uploads.github.com
# GitHub Assets & CDN (via GitHub's native IPv6)
2606:50c0:8000::133 objects.githubusercontent.com www.objects.githubusercontent.com release-assets.githubusercontent.com gist.githubusercontent.com repository-images.githubusercontent.com camo.githubusercontent.com private-user-images.githubusercontent.com avatars0.githubusercontent.com avatars1.githubusercontent.com avatars2.githubusercontent.com avatars3.githubusercontent.com cloud.githubusercontent.com desktop.githubusercontent.com support.github.com
2606:50c0:8000::154 support-assets.githubassets.com github.githubassets.com opengraph.githubassets.com github-registry-files.githubusercontent.com github-cloud.githubusercontent.com
```

### Steps

<Tabs>
<TabItem value="linux" label="Linux" default>

**1. Open `/etc/hosts`**

```bash
sudo nano /etc/hosts
```

**2. Add proxy entries at the end of the file**

Pick one of the options above (TransIP or DanWin1210) and paste at the bottom.

**3. Save and test**

```bash
# CTRL+X → Y → Enter (nano)

# Test connection
curl -I https://github.com 2>&1 | head -5
```

**4. Test git clone**

```bash
git clone https://github.com/Adekabang/docs-8labs.git
```

### Important Notes

- **SSH**: The proxy only works for HTTPS. For `git@github.com:...` (SSH), you still need an additional solution. Recommendation: use HTTPS remotes or set up an SSH proxy jump.
- **Rate limiting**: Public proxies may hit GitHub's rate limit faster. Use a PAT token for authenticated requests.
- **Long-term alternative**: Encourage GitHub to support native IPv6. In the meantime, this proxy is a practical workaround.

## Solution 2: Docker Hub via IPv6 Endpoint

Docker Hub provides a dedicated **IPv6 beta endpoint**:

```
registry.ipv6.docker.com
```

### Steps

**1. Log in to Docker Hub via the IPv6 endpoint**

```bash
docker login registry.ipv6.docker.com
```

Enter your Docker Hub username and password.

**2. Pull images using the IPv6 prefix**

```bash
# Official images (use the "library/" prefix)
docker pull registry.ipv6.docker.com/library/ubuntu:latest
docker pull registry.ipv6.docker.com/library/nginx:latest
docker pull registry.ipv6.docker.com/library/alpine:latest

# Non-official images
docker pull registry.ipv6.docker.com/storjlabs/watchtower:latest
```

**3. In Dockerfiles, use FROM with the prefix**

```dockerfile title="Dockerfile"
FROM registry.ipv6.docker.com/library/node:20-alpine
WORKDIR /app
COPY . .
RUN npm install
CMD ["node", "index.js"]
```

:::tip

The **`library/` prefix** is only for official images. For user images, the format is: `registry.ipv6.docker.com/<username>/<image>:<tag>`

:::

### Alternative: Set Up a Docker Mirror for IPv6

If you frequently build and pull many images, it's more efficient to set up a **Docker daemon mirror**:

```json title="/etc/docker/daemon.json"
{
  "registry-mirrors": ["https://registry.ipv6.docker.com"]
}
```

Then restart:

```bash
sudo systemctl restart docker
```

With the mirror configured, all `docker pull` commands automatically route through IPv6 without needing any extra prefix.

## Solution 3: NAT64 + DNS64 (Optional)

For a more universal solution (not limited to GitHub/Docker Hub), you can use public **NAT64/DNS64** services that automatically translate IPv4 addresses to IPv6.

### Using Google's DNS64

```bash
# Set DNS to Google NAT64
sudo nano /etc/resolv.conf
```

``` title="/etc/resolv.conf"
nameserver 2001:4860:4860::6464
nameserver 2001:4860:4860::64
```

With DNS64, every DNS query to an IPv4-only host is automatically wrapped with a NAT64 prefix, allowing your IPv6 VPS to reach the entire IPv4 internet.

:::warning

NAT64 only works for **outbound** connections. It cannot be used to expose a service from your VPS to the IPv4 internet.

:::

## Troubleshooting

| Issue | Solution |
|-------|----------|
| `git push` fails via HTTPS | Make sure you're using a Personal Access Token (PAT), not a password. Create one at GitHub → Settings → Developer settings → Tokens |
| SSH `git@github.com` still fails | Switch the remote to HTTPS: `git remote set-url origin https://github.com/user/repo.git` |
| Docker pull keeps timing out | Try `docker pull registry.ipv6.docker.com/library/<image>` directly |
| `curl` succeeds but `git` fails | Check `.gitconfig` — make sure no proxy settings conflict |
| Proxy is slow / times out | Public proxies are shared resources. Try an alternate proxy or NAT64 |
